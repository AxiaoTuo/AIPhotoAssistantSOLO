# 摄影初学者AI图片评价系统

## 项目愿景

将AI大模型打造成**"懂摄影、会教学、有耐心"的私人摄影导师**，帮助摄影初学者快速提升拍摄水平。

---

## 目标用户

| 用户群体 | 特征 | 核心需求 |
|---------|------|---------|
| 摄影爱好者 | 刚入门，有相机或手机 | 想知道自己拍得怎么样 |
| 自媒体创作者 | 需要提升图片质量 | 快速获得改进建议 |
| 学生党 | 学习摄影课程 | 课后练习需要反馈 |

---

## 核心使用场景

### 场景1：日常练习反馈
> 小明刚学摄影，周末去公园拍了几张花，想知道拍得怎么样。
>
> 他上传照片 → AI 3秒给出评分和点评 → 告诉他"构图不错，但曝光过度，建议减1档曝光补偿"

### 场景2：作业自检
> 摄影班布置了"光影"主题作业，交作业前想先自己检查一下。
>
> 上传作品 → AI 从四个维度打分 → 指出"光影运用得分较低，主体缺少明显的光影对比"

### 场景3：进步追踪
> 坚持练习一个月后，想看看自己有没有进步。
>
> 打开历史记录 → 查看评分趋势 → 发现构图分从65提升到了78

## AI 评价体系设计

### 四维度评分模型

```
        技术 (Technical)
             100
              │
              │
   叙事 ──────┼────── 构图
 (Narrative)  │    (Composition)
              │
              │
         美学 (Aesthetic)
```

### 评分标准参考

| 分数区间 | 水平描述 | 典型特征 |
|---------|---------|---------|
| 90-100 | 优秀 | 可参赛/发表级别 |
| 80-89 | 良好 | 有明确个人风格 |
| 70-79 | 中等 | 基础扎实，有亮点 |
| 60-69 | 及格 | 初学者正常水平 |
| 60以下 | 需改进 | 存在明显问题 |

### Prompt 设计理念
- **鼓励为主**：先肯定优点，再指出不足
- **具体可操作**：建议要能立刻实践
- **因材施教**：根据照片水平调整点评深度

## 成本预估

### AI API 调用成本

| 模型 | 单次分析成本 | 说明 |
|------|-------------|------|
| DeepSeek | ~¥0.01 | 性价比首选 |

### 月度成本估算（个人使用）
- 每日分析 10 张 × 30 天 = 300 次
- 使用 DeepSeek：约 ¥3/月
- 使用 GPT-4V：约 ¥30/月
